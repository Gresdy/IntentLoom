# IntentLoom E2E Runbook (T8)

> Scope: 5 critical user paths through the multi-CLI cockpit.
> Gates: `npm run typecheck` + `npx vitest run` + `cd src-tauri && cargo test --lib`.
> Status at the time of writing: all three gates are green
> (see the commit log on `main` from `bd13fcc` onwards).
> This document is the manual checklist an operator runs in
> a real Tauri build before tagging a release.

---

## 0. Pre-flight (3 commands, 3 minutes)

```bash
npm install                    # only when package.json changed
cd src-tauri && cargo build    # only when Cargo.toml changed; first run is slow
cd .. && npm run tauri dev     # boots Vite + opens the Tauri shell
```

Confirm the window opens at 1400x900, the top tab strip renders
six tabs (Claude / Codex / Gemini / OpenCode / OpenClaw / Hermes),
and the StatusBar model selector shows the current CLI as the
default trigger label.

If any tab shows a red "未安装" pill, the matching CLI is not on
`PATH`; the cockpit still launches, the rest of the runbook
applies to the tabs that are installed.

## 1. Switch CLI + send one prompt

**What this exercises**: top-tab -> `useModelStore.setCurrentApp`
-> `stream_ai` -> adapter `build_stream_command` -> CLI child process
-> `ai-stream-chunk` listener -> `appendContent` -> transcript.

Steps:
1. Click the **Claude** tab.
2. Type "hi" in the composer. Press Enter.
3. Wait for a real reply (or a streaming chunk from a sandbox CLI).
4. Confirm the response appears as an assistant message with the
   provider name visible in the message header.

**Pass criteria**: the messageStore now has 2 entries (user +
assistant), `useModelStore.currentApp === "claude"`, and the
StatusBar's elapsed-time counter incremented during the turn.

## 2. Switch to Hermes + send a prompt

**What this exercises**: the only adapter whose wire format is
plain text, so this path is the one most likely to expose a
parser regression (T5 / T6).

Steps:
1. Click the **Hermes** tab.
2. Type "ping" in the composer. Press Enter.
3. Confirm the first chunk `session_id: <id>` is **not** rendered
   into the transcript (T5 stripping). Use the dev-tools
   Network/event log if you need to confirm.
4. If Hermes is not authenticated, the stream should end with a
   red notice banner (notice--error), not folded into the
   assistant text (T6).

**Pass criteria**: no `session_id:` line in the transcript, and
any auth-failure block renders as a red notice, not as plain
text.

## 3. Switch to Gemini + tweak mode

**What this exercises**: the per-CLI mode + reasoning dropdowns
in the composer; verifies the spec for Gemini is wired (T3 +
`useComposerPrefsStore`).

Steps:
1. Click the **Gemini** tab.
2. In the composer, open the **Mode** dropdown. Pick "Plan".
3. Open the **Reasoning** dropdown. Pick a level.
4. Type "describe yourself" in the composer. Press Enter.
5. Confirm the CLI was launched with the matching flag (the
   `useComposerPrefsStore.modeFor('gemini')` was forwarded into
   `stream_ai`, which passed it to the gemini adapter's
   `build_stream_command`).

**Pass criteria**: a turn ran end-to-end without the dropdown
silently reverting. The dropdown selection should survive a tab
switch and come back when you re-select Gemini.

## 4. Cancel a long turn

**What this exercises**: the real-cancel path added in `bf8f9c5`:
`AiProcessRegistry` (session_id -> pid) + `cancel_ai` calling
`libc::kill(SIGTERM)` (Unix) / `taskkill /F /T /PID` (Windows).

Steps:
1. On any tab, type a prompt that produces a long turn
   ("explain every line of `ls` output", or similar).
2. While the spinner is up, click the **Cancel** button on the
   composer.
3. Confirm the spinner stops within ~1s (the controller flips
   `isStreaming` synchronously; the OS signal takes a few hundred
   ms).
4. Confirm the transcript ends at the last completed chunk, not
   at an empty assistant message.

**Pass criteria**: no orphan streaming flag, no duplicate
"finished" toast, and `AiProcessRegistry` no longer has the
session id (we can confirm by sending a second turn on the same
session — the registry lookup should not leak the old pid).

## 5. Rename a session from the History drawer

**What this exercises**: `useReasonixController.renameSession` ->
`useConversationStore.updateConversation` -> `persist` middleware
-> localStorage. T2.

Steps:
1. Open the History drawer (left rail -> clock icon, or
   `Ctrl+1`-`Ctrl+9` cycles to a prior session).
2. Hover over a session row. Click the pencil icon. The inline
   editor opens.
3. Type "T8 manual rename". Press Enter.
4. Close and reopen the drawer. The new title is still there.
5. Fully quit and relaunch the app. The renamed session still
   shows the new title (localStorage round-trip).

**Pass criteria**: the new title is stable across drawer
re-renders and across a full app restart.

## 6. Pick a workspace folder

**What this exercises**: `commands::projects::pick_workspace`
(Rust dialog plugin) -> `invoke("pick_workspace")` -> `state.meta.cwd`
-> StatusBar / TopBar. T1.

Steps:
1. Click the **open folder** icon in the StatusBar (or the
   Command Palette -> "Open Folder").
2. Native folder picker opens. Choose any directory.
3. Confirm the StatusBar's cwd pill shows the absolute path (or
   a truncated form like `.../path/to/folder` if longer than 40
   chars).
4. Restart the app. The cwd is restored from `localStorage`.

**Pass criteria**: cwd is visible in the StatusBar, persisted
across restarts, and a cancel on the picker leaves the previous
cwd untouched (not blanked out).

## 7. Trigger a send failure

**What this exercises**: T7's toast + notice pipeline.

Steps:
1. Stop the Hermes CLI externally (e.g. `pkill -f hermes` or
   rename its binary out of `PATH`).
2. Click the Hermes tab. Type anything. Press Enter.
3. Within ~1s, the composer spinner stops, a red toast appears
   top-right ("发送消息失败: ..."), and a red notice banner
   lands in the transcript (T6 + T7).

**Pass criteria**: error is visible in two places (toast +
banner) and the assistant message is **not** polluted with a
`\n\n错误: ...` line.

## 9. Known sandbox limits in this workspace

The Tauri shell itself needs a real display + native WebView,
which the CI / Codex sandbox does not have. The gates above
cover the front-end logic and the Rust commands that are
already unit-testable. The remaining 5 % is "does the
`tauri-plugin-dialog` actually pop a folder picker on a real
desktop" — that needs a human in the loop and is what this
runbook is for.
